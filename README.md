# router — Установка Tailscale на OpenWrt с podkop

Скрипты для прошивки роутеров OpenWrt, на которых одновременно работают **podkop** (обход блокировок через VLESS/Reality) и **Tailscale** (VPN-меш для удалённого доступа).

---

## Проблема, которую решает этот репозиторий

Схема сети: **Интернет → Провайдерский роутер → OpenWrt роутер → LAN**

После обновления Tailscale до версии 1.96.x на роутерах начались постоянные обрывы соединения каждые 5–7 секунд. Оказалось, что у новой версии есть баг с сохранением состояния (state-файл оставался пустым, nodekey генерировался заново при каждом переподключении).

Помимо этого, работа двух инструментов одновременно создаёт конфликт: podkop туннелирует трафик через серверы Cloudflare, Hetzner, DigitalOcean, OVH и других провайдеров — а серверы управления Tailscale (controlplane.tailscale.com) живут именно там. В итоге Tailscale пытался достучаться до своего controlplane через собственный же туннель, что приводило к петле и обрывам.

Ещё один источник проблем — Tailscale по умолчанию перехватывает DNS и прописывает свой сервер `100.100.100.100`. Этот DNS-запрос тоже шёл через туннель podkop, и при обновлении списков маршрутов podkop'ом соединение рвалось.

---

## Решение

1. **Откат на Tailscale v1.92.5** — стабильная версия без бага с state-файлом.
2. **`--accept-dns=false`** — запрет на перехват DNS, чтобы Tailscale не лез со своим резолвером через podkop.
3. **Прямой маршрут для `192.200.0.0/24`** — подсеть серверов controlplane.tailscale.com выводится напрямую, минуя туннель podkop.
4. **`tailscale serve`** — проброс портов 22, 80, 443 для удалённого доступа к роутеру через Tailscale-сеть.
5. **`/etc/rc.local`** — все фиксы применяются автоматически после перезагрузки.

---

## Установка

Выполни на роутере одну команду:

```sh
wget -O /tmp/s.sh https://raw.githubusercontent.com/vasneverov/router/main/small-tailscale.sh && sh /tmp/s.sh
```

Скрипт сам определит архитектуру роутера, скачает нужный пакет, запустит демон, попросит авторизоваться в Tailscale и применит все необходимые фиксы.

---

## Совместимость

- OpenWrt (ядро 5.x / 6.x)
- Архитектуры: aarch64, mips, x86 и другие (определяется автоматически)
- podkop v0.7+ (GuNanOvO репозиторий)
- Tailscale v1.92.5 (из репозитория [GuNanOvO/openwrt-tailscale](https://github.com/GuNanOvO/openwrt-tailscale))

---

## Что делает скрипт

1. Определяет архитектуру роутера через `opkg print-architecture`
2. Скачивает и устанавливает Tailscale v1.92.5
3. Запускает демон `tailscaled`
4. Запускает `tailscale up --accept-dns=false --accept-routes --reset` и выводит ссылку для авторизации
5. Добавляет прямой маршрут к подсети controlplane (`192.200.0.0/24`)
6. Включает `tailscale serve` на портах 22, 80, 443
7. Прописывает автозапуск всех фиксов в `/etc/rc.local`
